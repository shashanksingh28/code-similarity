Codereco

Introduction

The system is designed to evaluate a new recommendation algorithm as compared to a text based algorithm. For every time you search, you will be shown 5 recommendations from both algorithms, randomly shuffled (not ranked). Please rate the recommendations based on how much they helped you complete your task. Note that they need not give you exact solutions, they can give high level ideas or hints.

Please rate at least 15 recommendations per problem to submit and proceed to the next problem. Do not use search engines as that will impact the study.

Example

Try solving the problem in the test tab in the navigation bar. While you are at it, explore the voting interface and observe the changes in recommendations based on feature weights. As an example, see below screenshots

  1. Type in the method outline that you are looking for and hit 'search'. You will see some recommendations come up


  2. From the recommendations on the right, you found one to be somewhat related to conversion between values so gave it 2 stars. As soon as you voted, notice your updated vote count out of 15. Once this crosses 15, submit button will be enabled.


  3. With the hope of getting some related methods, you start writing comments and increase your 'Language' feature weights because you want to focus on matching comments. This helps you get a couple more relevant recommendations to vote for.


    1. Type in comments
    2. Increase weight for Language features
    3. Recommendation involving 'convertion'
    4. Recommendation involving 'integer'.

  4. Integer.toString() looks like it might do the job but it returns a String while you need to return an int. So you change focus to 'Structure' and 'Concept' features more and reduce weight on 'Language'. This helps you find Integer.parseInt(), which seems more like what you are looking for..

    1. Increase Structure (syntactical matches)
    2. Increase Concepts (conceptual matches)
    Reduce Language back

Problem 1 and 2

Note that sometimes you do not find exactly what you are looking for. In such cases, you should try writing something on your own and search after a couple of lines so that the algorithm has more data to recommend on. Remember the objective is to vote for recommendations that are relevant and helpful samples, not exact.
Also note that you have to write only ONE method per problem. Do not split your algorithm into two methods.
